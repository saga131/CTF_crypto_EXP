- # CTF_crypto_EXP

	> **这是一个“随时能重新上手”的 CTF Crypto 工作仓库。**
	>
	> 当我因为学习/比赛/生活中断刷题很久，
	>  当我忘了套路、忘了流程、甚至忘了自己会什么，
	>  **我只需要打开这个仓库，从这里重新开始。**

	------

	## 🧭 当你“断片很久”时，请从这里开始

	### 如果你现在脑子是空的，请只做这 3 步（不多想）

	**Step 1：进入 writeups/**

	```
	CTF_crypto_EXP/writeups/
	```

	**Step 2：新建一个题目目录**

	```
	writeups/<年份_比赛>/<题名>/
	```

	**Step 3：只写两样东西**

	- `README.md`（哪怕 5 行）
	- `exp.py` 或 `exp.sage`（哪怕是抄 WP）

	👉 **到此为止就算“重新开始成功”**
	 👉 不需要整理、不需要抽象、不需要写笔记

	------

	## 🚫 断片时你“绝对不要做”的事

	在你重新刷到 **20 题之前**，**请刻意忽略这些目录**：

	- `notes/`
	- `tools/`
	- `templates/`
	- `docs/`

	它们不是现在给你用的，
	 而是 **等你自然想用时再用的“仓库记忆”**。

	## 🗂️ 仓库真正的结构逻辑（给清醒时的你）

	```
	CTF_crypto_EXP/
	├─ writeups/        # 我做过 / 复现过的【具体题目】
	│                  # ——断片时唯一入口
	│
	├─ notes/           # 当我发现“这类题我见过很多次”时
	│                  # 才会写的【知识总结】
	│
	├─ tools/           # 当我烦了、不想再重复写代码时
	│                  # 才会自然产生的【工具】
	│
	├─ templates/       # 为了让我“少思考”的模板
	│
	├─ data/            # 公共数据
	│
	└─ docs/            # 规范（不是任务）
	```

	> **writeups 是入口**
	>  notes / tools 是结果
	>  **顺序不能反**

	------

	## ✍️ writeups/README.md 的最低标准（重要）

	每道题你只需要写这些内容：

	```
	## 题型
	RSA / ECC / Lattice / Polynomial / ？
	
	## 识别点
	（我当时是怎么认出这类题的）
	
	## 思路
	一句话：用什么方法
	
	## 状态
	✔ 已复现 / ❌ 未完全复现
	```

	**写不出来也没关系，写“未完全复现”是合法的。**

	------

	## 🧠 notes/ 是什么时候用的？

	**不是每题用。**

	只有当你出现这种感觉时：

	> “这已经是我第 3 次看到这种结构了。”

	你才去：

	```
	notes/knowledge-system/<rsa|ecc|lattice|prng_stream>.md
	```

	加 **3~5 行总结**。

	**notes 是“奖励”，不是任务。**

	------

	## 🔧 tools/ 是怎么产生的？

	tools 不是“设计出来的”，而是：

	> **“我已经不想再写这段代码了。”**

	那一刻你才把它挪进 `tools/`。

	如果你现在觉得 tools 很空：
	 **说明你在正确阶段。**
