## 一句先稳住你（非常重要）

> **这个框架不是“每天都要完整使用的”，
>  而是“99% 的时候你只用其中 1 个目录”。**

那个目录就是：

> 👉 **`writeups/`**

其余的 `notes / tools / templates`
 **不是“每天要用的东西”，而是“迟早会用的积累池”**。

- 现在就该用模板吗？

## 你现在真正该执行的规则（只保留 3 条）

### ✅ 规则 1（唯一入口规则）

> **任何“具体题目”，不管你是刷的、抄的、复现的，一律进 `writeups/`**

不判断、不纠结。

------

### ✅ 规则 2（最小记录规则）

> **每题只写 2 个文件：**

- `README.md`（哪怕 5 行）
- `exp.py / exp.sage`（能跑即可）

**不要求优雅、不要求抽象、不要求通用。**

------

### ✅ 规则 3（notes 是“事后奖励”，不是任务）

> **notes 不是每题都要写**

只有当你刷到：

- “啊？这又是同一类？”
- “这我第二次/第三次见了”

**才去 `notes/knowledge-system/xxx.md` 加 3 行总结。**

------

## 你现在应该“完全忽略”的东西（暂时）

在接下来 **10~20 题内**，你可以当这些目录不存在：

- `tools/`
- `templates/`
- `docs/`
- `data/`（除非题目真有附件）

👉 **它们现在的唯一作用：在那里等你。**

------

## 给你一个“绝对不会卡”的刷题流程（照抄）

### 假设你现在看到一道题（不管你会不会）

#### Step 1️⃣：只做这一条命令（或手动建目录）

```
writeups/2024_xxx_ctf/crypto_xxx/
```

#### Step 2️⃣：README.md 只写这 4 行

```
## 题型
RSA / ECC / Lattice / ？

## 识别点
（例如：同模数 / 小 e / 多项式）

## 思路
一句话：用什么

## 结果
OK / 未完全复现
```

> **不会写也没关系，写“未完全复现”完全合法。**

#### Step 3️⃣：exp.py

- 能跑就行
- 不抽象
- 不复用
- 就算是抄 WP 也没问题

**到此结束。立刻停。**
